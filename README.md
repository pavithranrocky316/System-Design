# 🚀 4-Week System Design Game Plan

A focused roadmap to build strong system design fundamentals and crack interviews.

---

## 📅 Week 1 — Foundations

> The building blocks that appear in every system design question.

### 🔹 Scaling
- Horizontal vs Vertical scaling
- Stateless vs Stateful services
- Auto-scaling basics

### 🔹 Load Balancing
- Algorithms: Round Robin, Least Connections, IP Hash
- L4 vs L7 load balancers
- Health checks

### 🔹 Caching
- Where to cache: Client, CDN, Server, DB
- Cache eviction policies: LRU, LFU
- Cache invalidation strategies

### 🔹 Databases
- SQL vs NoSQL
- Indexing basics
- Replication (Master-Slave)
- Sharding fundamentals

### 🔹 CAP Theorem
- Consistency vs Availability
- Partition tolerance
- Real-world trade-offs

---

## 📅 Week 2 — Core Components

> The tools you'll reach for in almost every design.

### 🔹 CDNs
- Push vs Pull model
- Edge locations
- Use cases

### 🔹 Message Queues
- Kafka vs RabbitMQ
- Async vs Sync communication
- Pub/Sub model

### 🔹 API Design
- REST principles
- Rate limiting
- API Gateway

### 🔹 Consistent Hashing
- Load distribution
- Use in caches and distributed DBs

### 🔹 Storage Systems
- Blob storage
- Object storage (e.g., S3)
- File systems

---

## 📅 Week 3 — Design Practice (Classic Systems)

> Apply Week 1 & 2 concepts to real-world problems.

### 🔹 URL Shortener
- Hashing
- Database design
- Scalability considerations

### 🔹 Rate Limiter
- Algorithms: Token Bucket, Leaky Bucket
- Distributed rate limiting

### 🔹 Social Media Feed (Twitter/Instagram)
- Fan-out on write vs read
- Feed ranking
- Social graph

### 🔹 Messaging System (WhatsApp)
- WebSockets
- Message delivery guarantees
- Storage design

### 🔹 Video Platform (YouTube)
- Video upload & processing
- CDN delivery
- Metadata storage

### 🔹 Distributed Systems

### 🔹 Search Systems

### 🔹 Notification Systems

### 🔹 Mock Interviews

---

## 🎯 How to Use This Plan

- Spend **2–3 hours daily**
- Focus on **understanding + articulation**, not just reading
- Practice **explaining designs out loud**
- Revisit weak areas every weekend

---

## 🧠 Pro Tip

> “System design is not about the perfect system — it's about making the right trade-offs.”

---

## 📌 Goal

By the end of 4 weeks, you should be able to:
- Break down any system design problem
- Identify key components quickly
- Justify trade-offs clearly
- Design scalable, production-ready systems
